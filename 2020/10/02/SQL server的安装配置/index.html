SQL的安装配置问题

创建于 2020-10-02 更新于 2020-10-31 阅读次数： Valine： 本文字数： 2.9k 阅读时长 ≈ 3 分钟

这个学期由于要学数据库原理，上课的时候要用到SQL server，所以花了一点时间研究了SQL server的安装与配置问题

什么是SQL server

Microsoft SQL Server(微软结构化查询语言服务器)是由美国微软公司所推出的关系数据库解决方案，最新的版本是SQL Server 2019，已在美国时间2019年11月3日发布。
数据库的内置语言原本是采用美国标准局和国际标准组织所定义的SQL语言，但是微软公司对它进行了部分扩充而成为作业用SQL。
几个初始版本适用于中小企业的数据库管理，但是近年来它的应用范围有所扩展，已经触及到大型、跨国企业的数据库管理。

SQL server是一种关系型数据库管理软件，是由微软公司开发的，所以对于很多使用MAC的朋友，你们是无法直接去安装SQL server，当然也可以安装，我也发现三种使用方法，后面在说吧，我现在使用的是Windows 10的电脑。

安装SQL server

SQL server的安装可以直接去官网下载,也可以直接点击进入下载页面，官网提供了两个版本的下载分别是Developer版本和EXpress版本供使用下载。

2017版直接下载链接
https://go.microsoft.com/fwlink/?linkid=853016

两个版本的区别

那么这两个版本有什么区别呢：

  • Developer版本

这是官网对这个版本的描述，也就是说这个版本是一个完全免费的版本，这个版本可以用来作为一个卡法和测试的工具，但是不能用来生产环境

  • Express版本

SQL Server 2019 Express 是免费的SQL Server 版本，这个版本很适合用来开发和生产桌面、Web 与小型伺服器应用程式，所以这个版本是可以用在生产环境的

对于这两者功能上的其他区别官网给了一种对比图，这是2017版的两个版本的区别

至于使用的功能上其实两个版本是没有什么区别的，作为学生使用下载两个应该版本都可以，官网上提供的是SQL server 2019的，但是这个版本是一个测试版本，我这里下载的是2017版的，这个版本的Developer版，点击上面的直接下载链接进入的也是2017版的下载界面。

下载步骤

点击下载2017的之后，出现了下面的界面：

这里可以进行一个选择，对于学生来说一般选择基本的就是帮你按默认的配置好了SQL server搜索引擎，当然选自定义的也可以，不过麻烦一点，我这里选择的是基本（B）

稍等片刻

接受协议，继续稍等片刻

选择安装路径，一般来说C盘空间够用的话，可以使用默认位置，点击安装

迎来漫长的等待

终于安装完成了，这个时候可以先不要关掉重新启动，倒不是说不可以关掉，其实网上说这个时候关机重新启动后面的无法配置了什么的，经过我的测试其实就算这个时候关掉重新启动貌似也是可以的，就现在用起来来说，当然你们在安装的过程中现在没有必要关机重新启动，因为还要下载一个东西。

这个时候SQL server已经安装好了，但是这个时候操作SQL server是很不方便的，你要使用命令行操作，当然也可以操作，只是不太方便，这个时候就可以安装SSMS，SSMS是SQL server的一个客户端工具，其实也就是一个图形化界面，可以更好的管理数据库，可以不用命令行去创建一个表，去进行查询操作啊什么的，对于初级使用者是很重要的，我还不知道我们老师要不要我们安装这个东西，说不定他要我们命令行操作，应该不会，先安装再说吧，点击有下面的安装SSMS。如果你关掉了这个页面，也可以直接去官网点击下载SSMS。

工具的下面有一个下载SQL Server Management Studio (SSMS)

下载开始，又进入漫长的等待

安装完了之后直接完成，这个安装教简单。

SQL server 配置

该安装的都安装了，这个时候就开始配置

启动TCP/IP协议

  1. 点击菜单中的SQL server 2017配置管理器，启动配置管理器

打开之后，先点击SQL Server服务，看鼠标点的第一项是否启动，没有启动就右键打开启动。

然后点击SQL server 网络配置，将右边的TCP/IP和上面的Named Pipes没有启动的启动。

  1. 配置SSMS

打开菜单中的Microsoft SQL Server Management Studio 18

打开Microsoft SQL Server Management Studio 18，第一次打开可能有点慢

打开之后进入登入界面

这个登入界面的服务器名称第一次登入可以填入计算机名称，或者直接输入localhost，计算机名字可以从下面找。

打开文档，右击计算机（我的电脑是此电脑）下面有一个属性，这里看不到，点击属性,然后高级系统设置 > 计算机名。就可以看到计算机名了，这里涉及一些东西就不发照片了。

填入计算机名字之后就可以点击连接了，理论可以直接使用了，这个时候你会发现你的数据库不用密码也可以登入，这时登入的是整个大的数据库系统，但是我们在使用的时候每一个用户可能都会建立一个具体的库，这是我们可以为这个具体的库设置登入密码与账号。

点击第一个，右击属性 > 安全性 > 连接

打开服务器身份验证的第二个选项，SQL server和Windows 身份验证模式，下面启用服务器代理账户可以设置账户密码

将连接界面的允许远程连接到此服务器打开

点击确定之后会要求你重新启动服务器，下面这个界面

鼠标处右键，有重新启动选项，点击重新启动

点击 安全性 > 登入名 > sa > 常规 > 状态 右键属性

上面是常规界面，可以设置密码，这就是为sa这个用户设置密码

上面是状态界面，将第一项授予，第二项启用

开始使用

创建一个数据库

上面设置好以后，就可以用你自己设置的账号进行登入了，在身份验证处选择使用SQL server验证，在下面填入你刚才设置的密码登入

登入之后，就可以创建一个具体的库

**点击 数据库 > 新建数据库 > 设置数据库的名字 > 确定 ，这里创建一个==SweepingRobot数据库==**，数据库名字下面还可以在下面进行具体的一些设置，这里就不设置了，简单的创建一个数据库。

在创建的SweepingRobot数据库下面可以创建一个表（关系）

数据库 > SweepingRobot > 右键表 > 新建表 > 表

在表中可以创建列，属性
